# Interview Guide

Deep product questions for gathering requirements.

## Core Questions

### Problem Discovery
- "What problem are we solving?"
- "Who feels this pain the most?"
- "How do users handle this today?"
- "What's broken about the current approach?"

### Solution Exploration
- "What should users be able to do?"
- "Walk me through the ideal experience"
- "What's the most important thing to get right?"

### Permissions
- "Which user roles can access this?"
- "What can each role do vs. not do?"
- "What happens if someone without access tries?"

### Edge Cases
- "What could go wrong?"
- "What if there's no data to show?"
- "What happens if the user makes a mistake?"

### Success
- "How will we know this worked?"
- "What metric should improve?"

## Tips

- Ask one question at a time
- Use context to skip questions you already know
- Prefer specific questions over open-ended ones
- Listen for unstated assumptions
